PYRAMID READINESS DAILY WORKING TEMPLATE

OVERARCHING GOAL
Become a confident, operationally reliable Business Intelligence (BI) team contributor who can analyze data, validate logic, deploy models, communicate and collaborate with the team, and hand off work cleanly — using Pyramid, Codex, BI judgment, and team collaboration skills.

PURPOSE
Use this file as the fast daily companion to the full execution guide.
This file is optimized for doing the work, not for explaining the whole program.

USE RULES
1. At the start of the day, copy the DAILY WORK BLOCK into your note for the day.
2. Fill in the block before starting work.
3. Complete the checklist items for the matching day.
4. End the day only after recording the evidence artifact and validation result.
5. At the end of each week, complete the WEEKLY SCORECARD BLOCK.

FINAL DAILY RULES
[ ] Do not skip Readiness Summary.
[ ] Do not skip Immediate Next Step.
[ ] Do not skip evidence capture.
[ ] Do not skip validation.
[ ] Do not jump to a visual if the logic layer is unclear.
[ ] Do not rewrite model logic if the issue is only parameter, list, format, or presentation behavior.
[ ] Do not dump the full Pyramid catalog unless the task truly requires it.
[ ] Do not widen scope after a failed validation until the current slice is repaired or reclassified.
[ ] Do not rely on Codex to replace manual understanding of logic, deployment policy, or reviewer access.
