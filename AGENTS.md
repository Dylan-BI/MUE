# MUE Codex Instructions

## Project Shape

- MUE is a Business Intelligence education repo with three main areas:
  - `source/`: reference curriculum and scripts. Treat this as read-only unless the task explicitly asks to change curriculum source.
  - `action/`: learner workspace for notes, evidence, reports, proxy data, and the dashboard.
  - `review/`: reviewer workspace and shared review data.
- `action/dashboard/` is a static dashboard plus Python standard-library tooling. It does not require Node packages or Python package installation.
- The Git remote is `https://github.com/Dylan-BI/MUE.git`.

## Environment

- Use Python 3.9 or newer.
- Do not print, commit, or rewrite `action/dashboard/.env`. Use `action/dashboard/.env.example` for variable names and structure.
- `action/dashboard/data.json` is generated by `action/dashboard/build_data.py` and is intentionally ignored by Git.
- ChatGPT Desktop local-environment setup/actions can point at the scripts in `.codex/scripts/`.
- `.worktreeinclude` copies ignored local learner/config files into Codex-managed worktrees so dashboard tasks see the same working data.

## Common Commands

- Setup or refresh dashboard data:
  `powershell -ExecutionPolicy Bypass -File .codex/scripts/setup.ps1`
- Rebuild dashboard data:
  `powershell -ExecutionPolicy Bypass -File .codex/scripts/build-dashboard.ps1`
- Serve the review dashboard locally:
  `powershell -ExecutionPolicy Bypass -File .codex/scripts/serve-review-dashboard.ps1`
- Watch learner files and rebuild automatically:
  `powershell -ExecutionPolicy Bypass -File .codex/scripts/watch-dashboard.ps1`

## Workflow Rules

- After changing learner notes, evidence, reports, review data, dashboard code, or curriculum source that feeds the dashboard, run the dashboard build.
- Keep generated/proxy learner artifacts out of Git unless the user explicitly asks to commit them.
- Preserve the 28-working-day curriculum constraint. Use the existing scripts instead of manually creating daily note files when possible.
- For dashboard/reviewer work, prefer the local review server when reviewing interactive behavior; otherwise opening `action/dashboard/dashboard.html` is enough for static inspection.

## Verification

- Minimum verification for dashboard data changes:
  `python action/dashboard/build_data.py`
- For review-server behavior, start:
  `python action/dashboard/review_server.py --port 8080 --host 127.0.0.1`
